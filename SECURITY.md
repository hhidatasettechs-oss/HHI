# Security Policy — Hollow House Institute (HHI)

## Supported Versions

Only officially released, tagged versions of the Hollow House Institute (HHI) repository are supported for security review.

| Version | Supported |
|--------|-----------|
| v1.0.x | ✅ Yes |
| < v1.0 | ❌ No |

Unreleased branches, forks, and derivative works are not supported.

---

## Reporting a Vulnerability

HHI treats security, integrity, and misuse risks as high-priority governance issues.

If you discover a vulnerability, compliance concern, or misuse risk:

**DO NOT** open a public GitHub issue.  
**DO NOT** disclose details publicly or to third parties.

### Report privately via:
**Email:** security@hollowhouseinstitute.org

Include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Affected files, commits, or versions
- Any evidence or logs (redacted as necessary)

---

## Scope of Security Review

This security policy covers:
- Repository integrity
- Build and release provenance
- Governance enforcement mechanisms
- Licensing compliance infrastructure
- CI/CD workflows

This policy **does not** cover:
- User-generated derivative datasets
- Third-party forks
- Unauthorized redistributions
- Non-licensed deployments

---

## Enforcement

Confirmed security violations, misuse, or license breaches may result in:
- Immediate license revocation
- Legal enforcement
- Access termination
- Takedown actions

HHI reserves all rights under applicable law.
